适用于：photo.dbbrewbeer.com / photo.dbbrewbeer.com/admin
项目目标
做一个简洁、稳定、可快速上线的活动图片直播平台，支持摄影师上传、嘉宾浏览与下载。
核心域名
前台：photo.dbbrewbeer.com 后台：photo.dbbrewbeer.com/admin
核心存储
腾讯云 COS + 数据万象（CI）
当前版本范围
单活动、单后台账号、图片浏览与下载、自动生成浏览图和下载图

一句话定义：摄影师上传原图后，系统自动生成“前台浏览图”和“带水印高清下载图”；嘉宾打开首页即可按时间查看全部图片并下载。

1. 项目背景与目标
背景：现有活动需要一个简洁的图片直播工具，尽快上线，满足摄影师实时上传、嘉宾快速浏览和下载的需求。
目标：以最小可用范围完成第一版，不做复杂会员体系、不做复杂多角色权限、不做冗余运营功能，重点保证上传链路稳定、浏览快、下载体验清晰。
前台用户无需登录，打开首页即可看图。
摄影师通过后台登录后上传原图。
所有对外可见图片都必须带“时光酿造所”水印。
前台浏览图与下载图分离：浏览看 display，下载拿 download。

5. 核心功能需求
5.1 首页浏览
默认展示全部已发布图片。
按时间倒序，最新上传的照片排在最前。
支持点击查看大图预览。
支持点击下载按钮下载带水印高清图。
首版不要求复杂筛选与搜索。
5.2 摄影师登录
首版使用单账号密码登录。
账号密码通过环境变量管理，不写死在仓库代码中。
登录后写入 session，未登录无法访问后台。
5.3 摄影师上传
支持选择或拖拽上传原图。
建议支持批量上传。
上传过程中显示进度与结果。
原图上传到 originals 目录；上传完成后自动生成 display 与 download。

6. 存储结构与命名规范
目录结构：使用腾讯云 COS，按“资源类型 / 活动标识 / 日期 / 文件名”组织。
/originals/default/YYYY-MM-DD/{timestamp}-{random}.jpg
/display/default/YYYY-MM-DD/{timestamp}-{random}.jpg
/download/default/YYYY-MM-DD/{timestamp}-{random}.jpg
活动标识首版固定为 default，后续可扩展为具体活动名。
文件名建议采用“时间戳 + 随机串”，避免重名并利于追踪。
日期目录在上传时由代码自动生成，无需人工提前创建。

8. 上传与访问链路
摄影师访问 /admin，登录后台。
后台页面向服务端请求上传授权信息。
浏览器将原图直传到 COS 的 originals 目录。
在上传时或上传后，由数据万象生成 display 与 download 两份图片。
前台首页读取 display 列表并按时间倒序展示。
用户点击下载时，获取 download 图片。
关键设计原则：摄影师原图不经过 Vercel 中转处理，优先采用浏览器直传 COS，再由云端生成衍生图。这样可以避免大文件上传受限并减轻 Vercel 函数压力。

14. 风险与注意事项
若摄影师原图尺寸或体积过大，需验证数据万象处理链路是否满足实际活动图片体量。
若 download 目录做公开访问，需接受下载链接可被直接访问的风险；后续可升级为签名下载。
务必避免将 SecretId / SecretKey 提交到 GitHub；仅放本地 .env 与 Vercel 环境变量。
首版以单活动、单账号为主，后续若扩展多活动、多摄影师，需要增加元数据表与后台管理能力。
15. 验收标准
摄影师可以登录后台并成功上传原图。
系统可以生成 display 与 download 两份图片。
前台首页可按时间倒序展示已上传图片。
点击下载时，用户拿到的是带“时光酿造所”水印的高清下载图。
所有对外可见图片都带统一水印。
正式域名 photo.dbbrewbeer.com 可访问，后台入口为 /admin。
